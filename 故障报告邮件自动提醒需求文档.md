# 故障报告邮件自动提醒功能需求文档

## 1. 项目概述
基于 Google Apps Script 开发故障报告邮件自动提醒系统，实现故障报告的定期提醒和超期预警功能。

## 2. 数据源配置
- **Google Sheets ID**: `1YAPdZKVEOHgCGIJRQwWTQBmwaWIS4yd1SQKJJfRCtAU`
- **故障报告数据表**: `Failure_Database`
- **通知清单表**: `通知清单`

## 3. 功能需求

### 3.1 提醒对象筛选
- 从 `通知清单` 表中读取邮箱地址、车间、工序信息
- 基于 **车间** 和 **工序** 字段进行筛选（"与"的关系）
- 支持一个邮箱对应多个车间的情况（如：TB1/TB2 表示TB1和TB2车间）
- 当车间字段包含多个车间时，只要任一车间有未上传的故障报告，都会向该邮箱发送提醒
- 只向符合条件的邮箱发送提醒邮件
- 通知清单表结构：车间 | 工序 | 邮箱

### 3.2 提醒频率策略
- **正常情况**: 每周提醒一次
- **超期预警**: 从分配日期开始计算，超过7天未提交的故障报告每天提醒
- **定时执行**: 参考 `Changeover Process Para Verification` 项目的定时执行机制

### 3.3 故障报告关键字段
| 字段名 | 说明 | 用途 |
|--------|------|------|
| 编号 | 故障报告唯一标识 | 邮件内容显示 |
| 机台号 | 故障发生的机台 | 邮件内容显示 |
| 问题描述 | 故障详细描述 | 邮件内容显示 |
| 提交日期 | 故障提交时间 | 计算超期天数 |
| 车间 | 故障发生车间 | 筛选条件 |
| 工序 | 故障发生工序 | 筛选条件 |
| 故障报告编号 | 系统编号 | 邮件内容显示 |
| 分配日期 | 任务分配时间 | 超期计算基准 |
| 上传日期 | 附件上传时间 | 邮件内容显示 |
| 附件 | 相关附件 | 邮件内容显示 |

### 3.4 邮件内容要求
- 包含所有关键字段信息
- 显示距离分配日期已过去的天数
- 采用卡片式布局设计（参考现有项目样式）
- 中英文双语显示

### 3.5 邮件模板样式
- 参考 `Changeover Process Para Verification` 项目的卡片式布局
- 现代化设计风格
- 响应式表格布局
- 状态标识和颜色区分

## 4. 技术实现

### 4.1 核心功能模块
1. **数据获取模块**: 从 Google Sheets 读取故障报告和通知清单
2. **筛选逻辑模块**: 基于车间和工序筛选符合条件的记录，支持多车间匹配
3. **超期计算模块**: 计算距离分配日期的天数
4. **邮件生成模块**: 生成格式化的邮件内容
5. **定时触发模块**: 设置每周和每日的自动执行

### 4.2 执行策略
- **每周执行**: 检查所有故障报告状态
- **每日执行**: 检查超期故障报告（超过7天未提交）
- **智能去重**: 避免重复发送相同内容的邮件
- **定时机制**: 采用灵活的定时配置，支持按月份、星期、周内日期、具体日期等条件设置

### 4.3 错误处理
- 数据读取异常处理
- 邮件发送失败重试机制
- 日志记录和监控

## 5. 配置要求

### 5.1 Google Apps Script 权限
- 读取 Google Sheets 数据
- 发送 Gmail 邮件
- 创建定时触发器

### 5.2 数据表结构要求
- `Failure_Database` 表必须包含所有关键字段
- `通知清单` 表结构：车间 | 工序 | 邮箱
- 车间字段支持多车间表示（如：TB1/TB2，用斜杠分隔）
- 日期字段格式统一
- 分配日期字段用于超期计算（超过7天）
